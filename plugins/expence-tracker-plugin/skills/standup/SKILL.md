## Задача

Сформировать читаемый отчёт для утреннего стендапа на основе коммитов за вчерашний день.

## Контекст выполнения
Вчерашняя дата `date`: !`date -v-1d +%Y-%m-%d`
Коммиты за вчера 
```bash
!git log --after="<вчера> 00:00:00" --before="<сегодня> 00:00:00" \
--pretty=format:"%h %s%n%b" --author="$(git config user.name)"
```


## Алгоритм выполнения

1. Если коммитов нет — сообщить об этом и завершить.
2. Для каждого коммита разобрать:
   - `type(scope)`: определить категорию работы
   - описание: раскрыть смысл изменения (не просто повторить заголовок)
3. Сгруппировать по пакетам / областям:
   - `api` — бэкенд, эндпоинты, БД
   - `web` — фронтенд, UI, страницы
   - `shared` — общие типы
   - `chore/docs/refactor` — прочее
4. Вывести отчёт в формате ниже.

## Формат отчёта

```
Стендап — <дата в формате DD.MM.YYYY>
Автор: <git user.name>

Что было сделано вчера:

[Web]
• <человекочитаемое описание задачи 1>
• <описание задачи 2>

[API]
• <описание задачи 3>

[Инфраструктура / Документация]
• <описание задачи 4>

Итого: X коммит(а/ов)
```

## Правила

- Описание каждого пункта — на русском, полным предложением, понятным без контекста кода
- Не копировать заголовок коммита дословно — раскрывай смысл
- Если несколько коммитов относятся к одной фиче — объединяй в один пункт
- Технические детали (имена функций, файлы) — только если они важны для понимания
- Секции без коммитов — не выводить
